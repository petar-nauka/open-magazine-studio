# Magazine Studio — Функционалности и процес

> **English summary:** A web app that turns pasted/uploaded articles (Google Docs / Word `.docx`)
> into a consistent "БГ Наука" magazine layout (Montserrat + Andika, per-article accent colour,
> full-bleed photo opener, two-column body), organises them into issues, and exports PDFs via the
> browser's print-to-PDF. React + Vite + TypeScript front end; Supabase (Postgres + Storage) backend.
> Images are uploaded (compressed) to Supabase Storage; everything is keyed off the `mag_pdf_*` tables.

Документ на текущото състояние след Фаза 1 + Фаза 2.

---

## 1. Какво прави приложението

Превръща статия (от Google Docs или Word `.docx`) в **последователен дизайн на списание „БГ Наука"** и я подрежда в **броеве**. Целта: приложението прави 90% от оформлението по предварително зададени правила, а ти/ИИ само донастройвате.

**Работен поток:**
1. **Начало** (`/`) → виждаш последните броеве → „**+ Нов брой**" или отваряш съществуващ.
2. В **брой** (`/issue/:id`) → „**+ Нова статия**" → отваря работното поле.
3. **Импорт**: поставяш (Ctrl+V) от Google Docs или качваш `.docx`. Снимките се **смаляват и качват в Supabase Storage** автоматично (показва се прогрес).
4. **Преглед**: центърът веднага показва **БГ-Наука дизайна** (същото като PDF-а), обновява се при редакция.
5. **Запазваш** (toast „Запазено ✓"). Статията влиза в броя.
6. **Подреждаш** статиите в броя (стрелки ↑/↓).
7. **Теглиш**: всяка статия поотделно или **целия брой** като един PDF (през браузърния печат → „Запази като PDF").

## 2. Страници / маршрути

| Маршрут | Какво е |
|---|---|
| `/` | Начало — табло с последните 5 броя + „Нов брой" |
| `/issue/:id` | Брой — корица, подредени статии, добавяне, теглене (статия / цял брой) |
| `/new` | Работно поле за нова статия (paste/upload → БГ-Наука преглед → запис). `?issue=<id>` я вписва в броя |
| `/edit/:id` | Редакция на запазена статия (същият БГ-Наука преглед) |
| `/render` | Чистият БГ-Наука рендер за печат. `?id=<uuid>` (една статия), `?issue=<id>` (корица + всички статии). Лента „Свали PDF" |
| `/archive` | Плосък списък на всички статии/категории |
| `/settings` | Настройки с табове: **Брандиране** / **AI** |

**Навигация:** постоянен хедър на всяка страница — лого = Начало, + „Архив", „Настройки".

## 3. Дизайн система („БГ Наука")

- **Шрифтове:** Montserrat (заглавия) + Andika (тяло) — вградени в `public/fonts/`.
- **Бранд цвят:** `#007daa` + бяло. **Акцент на статия:** избор от 6 цвята (Тюркоаз/Магента/Синьо-зелен/Златен/Индиго/Зелен) + свободен цвят; пази се в `layout_config.accent`. Акцентът оцветява колонтитула, подзаглавията, инициала, булетите, линковете, цитатите.
- **Шаблони (в `public/magazine.css` + `src/magazine/`):** `opener` (цяла снимка + бяло заглавие), `body` (2 колони, цветен инициал, главни подзаглавия, булети с удебелено начало, цитати, снимки в потока), `references`.
- **Лога:** `public/brand/logo-01/03/09/12.png` (хоризонтално тъмно / бяло върху тъмно / икона / надпис).
- **Странициране:** Paged.js (A4, колонтитул горе, номер на страница долу).

## 4. Данни (Supabase, префикс `mag_pdf_`)

| Таблица | Роля |
|---|---|
| `mag_pdf_categories` | Броеве (name, issue_number, cover_image_url, **cover_pdf_url**) |
| `mag_pdf_articles` | Статии (title, author, **layout_config** вкл. `accent`, category_id, **sort_order**, tags, status) |
| `mag_pdf_content_blocks` | Блокове на статия (type, content = текст или **URL към снимка**, position, metadata) |
| `mag_pdf_issue_inserts` | Реклами в броя (image_url, kind, sort_order) — пълностранични реклами-снимки, подредени с общ sort_order със статиите |
| `mag_pdf_app_settings` | Конфигурация (ai_config, branding_config) |
| Storage bucket `mag_pdf_images` | Качените (компресирани) снимки; в БД се пазят само public URL-и |

**PDF/печат:** изцяло през браузъра — `/render` + Paged.js → Ctrl+P / „Свали PDF" → „Запази като PDF". Без сървърен експорт. Снимка-корица става първа страница на броя; PDF-корица се тегли отделно.

## 5. Настройка (за нов компютър / инстанция)

1. `npm install` в `magazine-pdf/`.
2. Създай `magazine-pdf/.env`:
   ```
   VITE_SUPABASE_URL=https://твоят-проект.supabase.co
   VITE_SUPABASE_ANON_KEY=anon-ключ
   ```
   (Имената трябва да започват с `VITE_`. Без кавички. Само anon ключ — не service_role.)
3. Пусни SQL миграциите в Supabase → SQL Editor (в реда на датите), от `supabase/migrations/`:
   - `..._mag_pdf_prefixed_schema.sql` (таблиците)
   - `..._storage_images_bucket.sql` (bucket `mag_pdf_images` + anon политики)
   - `..._add_cover_pdf.sql` (колоната `cover_pdf_url`)
   - `..._issue_inserts.sql` (таблицата `mag_pdf_issue_inserts` за реклами в броя)
4. `npm run dev` → http://localhost:5173 (или `-- --host` за достъп в мрежата).

**Команди:** `npm run dev` · `npm run build` · `npm run preview` · `npm run lint` · `npm run typecheck` · `npm test` · `npm run export:pdf -- <url> <out.pdf>` (Playwright експорт, опционален).

## 6. Какво е направено (Фаза 1 + 2)

- БГ-Наука рендер по подразбиране (Montserrat/Andika, opener, 2 колони, инициал, колонтитул) — еднакъв в редактора и в PDF-а.
- Импорт от `.docx`/Google Docs; **снимки → Supabase Storage** (компресирани), малки редове в БД.
- Запис/зареждане от БД; toast „Запазено"; прогрес при импорт.
- Избор на акцентен цвят на статия.
- Броеве: табло, страница на брой, **подреждане** на статии, **качване на корица** (снимка/PDF), теглене на статия и на цял брой.
- Навигация: постоянен хедър, настройки с табове.
- **Съдържание (TOC)**: авто-генерирана страница „Съдържание" в тегления цял брой — малка снимка + заглавие + номер на страница (номерата чрез Paged.js `target-counter`). Корица=стр.1 и съдържание=стр.2 се броят, но без изписан номер; пълноекранните снимки (opener/реклами) също без номер.
- **Реклами**: вмъкване на 1-странични реклами-снимки на произволно място между статиите (качват се в Storage; подреждат се със стрелки заедно със статиите; побират се цели — contain).

## 7. Известни ограничения / следващо (Фаза 2b)

- **Сливане на PDF-корица/реклами** в единия файл (сега PDF се тегли отделно; реклами-снимки вече се вграждат) — изисква pdf-lib.
- **Преглед на живо при всяка буква** — сега обновява при промяна (debounce ~0.7s, iframe).
- **Дубликати**: повторно записване на същата импортирана статия създава нов запис (няма автоматична проверка).
- **Bundle ~930KB** (Paged.js) — за публичен хостинг да се раздели (code-splitting).
- **RLS** е отворен за anon (MVP, един потребител) — да се стегне преди реално публикуване.
- `window.print()` минава през print диалога (не е „един клик"); по желание — локален Playwright експорт сървър.
